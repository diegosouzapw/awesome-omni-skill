# SBIR Grants Writing - 經濟部小型企業創新研發計畫撰寫指南

## Overview

SBIR (Small Business Innovation Research，小型企業創新研發計畫) 是經濟部推動的旗艦級創新補助計畫，旨在鼓勵中小企業進行技術與服務創新研發，提升產業競爭力。本技能提供完整的計畫書撰寫指引，從創新論述到市場分析，從經費編列到產業別策略。

**核心原則：SBIR 計畫書必須同時展現技術創新性、市場可行性、商業化潛力及產業效益。** 與學術研究計畫（如 NSTC）不同，SBIR 強調的是「從創新到市場」的完整路徑。

## When to Use This Skill

此技能適用於：
- 規劃 SBIR Phase 1/2/2+ 計畫書架構
- 撰寫創新論述與技術說明
- 進行市場分析與競爭優勢論述
- 規劃商業化策略與營收預估
- 編列經費與撰寫預算說明
- 準備審查簡報（Phase 1）
- 回應審查委員意見
- 了解不同產業別的申請策略
- 避免常見的撰寫錯誤

## SBIR vs NSTC 關鍵差異

理解 SBIR 與 NSTC 的本質差異是成功的第一步：

| 維度 | SBIR (經濟部) | NSTC (國科會) |
|------|--------------|--------------|
| **目標對象** | 中小企業 | 學術研究人員 |
| **核心目標** | 技術創新 → 商業化 | 科學突破 → 學術貢獻 |
| **評審組成** | 業界專家為主 | 學界專家為主 |
| **評審重點** | 市場潛力、產業效益 | 學術創新、研究方法 |
| **文獻探討** | 簡要，聚焦技術差異化 | 深入完整 |
| **市場分析** | **核心必要** | 不重要 |
| **商業模式** | **核心必要** | 不需要 |
| **預期成果** | 產品、營收、技術移轉 | 論文、專利、學術影響力 |
| **團隊介紹** | 企業實績、產業經驗 | 學術發表、研究經歷 |
| **經費編列** | 含自籌款規劃 | 全額補助 |

**關鍵心態轉換**：SBIR 審查委員問的是「這能不能賣？能賣多少？對產業有什麼貢獻？」而非「這在學術上有什麼突破？」

## SBIR 計畫階段概覽

### Phase 1：先期研究（先期規劃）

**目標**：驗證創新構想的可行性

| 項目 | 內容 |
|------|------|
| **補助上限** | 150 萬元（2026年起調高） |
| **期程** | 6 個月 |
| **補助比例** | 最高 50%（需自籌 50%） |
| **審查方式** | 簡報審查為主 |
| **重點** | 創新構想、問題定義、初步可行性驗證 |

**適合對象**：
- 有創新想法但尚未驗證可行性
- 需要進行初步技術研究或市場驗證
- 首次申請 SBIR 的企業

### Phase 2：研究開發（細部計畫）

**目標**：進行產品雛形開發、小量試產或服務試營運

| 項目 | 內容 |
|------|------|
| **補助上限** | 1,200 萬元（2026年起調高） |
| **期程** | 6-24 個月 |
| **補助比例** | 最高 50% |
| **審查方式** | 計畫書審查 + 簡報審查 |
| **重點** | 技術可行性、產品開發、市場驗證、商業化規劃 |

**適合對象**：
- 已完成 Phase 1 或已有初步可行性驗證
- 準備進入產品開發階段
- 有明確的商業化路徑

### Phase 2+：加值應用

**目標**：將研發成果商業化、規模化

| 項目 | 內容 |
|------|------|
| **補助上限** | 600 萬元（2026年起調高） |
| **期程** | 依計畫而定 |
| **重點** | 商業化、市場拓展、營收成長 |

**適合對象**：
- 已完成 Phase 2 並有初步產品
- 準備進入市場推廣階段
- 需要資金支持規模化

## 申請資格

### 基本資格

符合「中小企業認定標準」之國內公司、有限合夥或商業登記：

1. **實收資本額**：新台幣 1 億元以下，**或**
2. **員工人數**：經常僱用員工數未滿 200 人

### 排除條件

以下情況不得申請：

- ❌ 有陸資投資
- ❌ 外國營利事業在台設立之分公司
- ❌ 近 5 年內有政府計畫重大違約
- ❌ 近 3 年內有欠稅紀錄
- ❌ 嚴重違反環保、勞工、食安等法規
- ❌ 銀行拒絕往來戶

### 同時申請限制

- 每家企業同一時間執行中的 SBIR 計畫以 **2 案** 為限
- 申請第 2 案需前案評分達 **88 分以上**

## 計畫書核心架構

### Phase 1 計畫書/簡報結構

Phase 1 通常以「簡報審查」為主，內容需涵蓋：

1. **公司簡介** (1-2 頁)
   - 公司基本資料
   - 團隊核心能力
   - 相關研發經驗

2. **問題陳述與創新構想** (3-5 頁)
   - 要解決的問題/客戶痛點
   - 現有解決方案的不足
   - 你的創新構想
   - 技術/服務創新點

3. **可行性評估規劃** (2-3 頁)
   - 如何驗證可行性
   - 研究/測試方法
   - 預期驗證成果

4. **市場初步分析** (1-2 頁)
   - 目標市場
   - 潛在客戶
   - 市場規模估計

5. **時程與經費** (1 頁)
   - 6 個月工作時程
   - 經費編列概要

### Phase 2 計畫書結構

Phase 2 需要完整的計畫書，結構如下：

1. **公司概況** (2-3 頁)
   - 公司基本資料與沿革
   - 組織架構與股東結構
   - 營運與財務狀況
   - 過去研發成果與專利
   - 政府計畫執行實績

2. **計畫目標與創新性** (5-8 頁)
   - 計畫背景與問題定義
   - 研發目標（具體、可量化）
   - 創新性論述（與現有技術比較）
   - 技術指標與規格

3. **研發方法與執行策略** (8-12 頁)
   - 研發方法與技術路徑
   - 工作項目分解
   - 各階段查核點
   - 潛在問題與對策
   - 甘特圖

4. **市場分析與商業化規劃** (5-8 頁)
   - 目標市場分析 (TAM/SAM/SOM)
   - 競爭分析
   - 價值主張
   - 商業模式
   - 營收預估

5. **團隊能力** (2-3 頁)
   - 計畫主持人資歷
   - 核心團隊成員
   - 外部合作夥伴

6. **經費編列** (2-3 頁)
   - 各類費用明細
   - 費用說明與理由

7. **預期成果與產業效益** (2-3 頁)
   - 量化成果指標
   - 經濟效益
   - 產業影響

## 審查標準

### 核心評審維度

1. **創新性 (Innovation)**
   - 技術/服務是否具有創新性？
   - 與國內外現有技術相比有何突破？
   - 是否解決了真實的市場問題？

2. **可行性 (Feasibility)**
   - 技術方法是否可行？
   - 時程規劃是否合理？
   - 團隊是否有能力執行？

3. **市場潛力 (Market Potential)**
   - 目標市場規模是否足夠大？
   - 是否有明確的客戶需求？
   - 競爭優勢是否可持續？

4. **產業效益 (Industry Impact)**
   - 對產業升級的貢獻？
   - 經濟效益（營收、就業）？
   - 技術擴散效果？

5. **團隊能力 (Team Capability)**
   - 主持人資歷與經驗
   - 團隊專業互補性
   - 過去執行實績

6. **經費合理性 (Budget Appropriateness)**
   - 經費編列是否合理？
   - 是否與工作內容相符？
   - 自籌款規劃是否可行？

### 評分門檻（工程司參考）

| 分數 | 等級 | 說明 |
|------|------|------|
| 92+ | Top 5% | 傑出研究獎候選 |
| 88+ | Top 15% | 可申請第 2 案 |
| **81+** | **Top 55%** | **通過門檻** |
| 80 以下 | - | 不建議核定 |

> ⚠️ 評分標準可能因領域而異，請以官方公告為準

## 經費編列

### 經費類別與限制

| 類別 | 說明 | 建議比例 |
|------|------|----------|
| **人事費** | 計畫相關人員薪資 | ≤ 60% |
| **耗材費** | 實驗材料、軟體授權等 | 依需求 |
| **設備使用費** | 設備租賃或使用費 | 依需求 |
| **技術引進費** | 技術授權、顧問費 | 依需求 |
| **委託研究費** | 外包研發項目 | ≤ 30% |
| **國外差旅費** | 國際會議、合作交流 | 7-10 萬 |
| **其他費用** | 文獻、專利申請等 | 依需求 |

### 經費編列原則

1. **每筆費用都要有理由**
   - 與計畫工作項目對應
   - 說明為何需要、如何計算

2. **避免過高的單一項目**
   - 大額設備需說明必要性
   - 說明為何無法使用現有設備

3. **人事費要合理**
   - 符合市場行情
   - 人力配置與工作量相符

4. **自籌款規劃要可信**
   - 說明資金來源
   - 展現執行決心

詳細指引請參考 `references/budget_preparation.md`

## 產業別差異化策略

SBIR 涵蓋六大產業領域，各有不同的審查重點：

### 1. 機械 (Machinery)

**常見主題**：智慧製造、自動化設備、精密機械
**審查重點**：
- 生產效率提升數據
- 國產化比例
- 設備規格與性能指標

### 2. 民生化工 (Chemical & Materials)

**常見主題**：新材料、綠色製程、循環經濟
**審查重點**：
- 環保與永續面向
- 製程創新
- 成本效益分析

### 3. 生技 (Biotechnology)

**常見主題**：醫療器材、新藥開發、檢測技術
**審查重點**：
- 法規路徑（FDA、TFDA）
- 臨床驗證計畫
- IP 策略

### 4. 服務 (Services)

**常見主題**：數位轉型、平台服務、商業模式創新
**審查重點**：
- 服務創新性
- 用戶體驗設計
- 營運模式可行性

### 5. 資通 (ICT)

**常見主題**：AI/ML、IoT、雲端服務、資安
**審查重點**：
- 技術架構
- 資料安全
- 擴展性

### 6. 電子 (Electronics)

**常見主題**：半導體、電子元件、系統整合
**審查重點**：
- 技術規格
- 供應鏈整合
- 國際競爭力

詳細指引請參考 `references/industry_specific_guidance.md`

## 撰寫策略重點

### 創新性論述

**Do ✅**
- 明確說明與現有技術的差異
- 提供比較數據（效率提升 X%、成本降低 Y%）
- 說明為何這個創新重要（解決什麼問題）
- 有初步驗證數據更佳

**Don't ❌**
- 只說「創新」但沒有具體說明
- 誇大不實的宣稱
- 忽略競爭技術的存在
- 抄襲現有技術

### 市場分析

**Do ✅**
- 使用 TAM/SAM/SOM 框架
- 引用可信的市場研究數據
- 說明具體的目標客戶
- 有潛在客戶的 LOI 加分

**Don't ❌**
- 只說「市場很大」
- 使用不可信來源
- 忽略競爭分析
- 沒有說明如何取得市場

### 執行規劃

**Do ✅**
- 工作項目分解明確
- 時程合理可行
- 有清楚的查核點
- 包含風險對策

**Don't ❌**
- 時程過於樂觀
- 工作項目模糊
- 沒有考慮潛在問題
- 查核點不具體

## 常見錯誤

1. **創新性不明確**
   - 沒有說清楚「新在哪裡」
   - 與競品的差異不明顯

2. **市場分析薄弱**
   - 只有「市場很大」的空泛描述
   - 沒有具體的客戶驗證

3. **時程過於樂觀**
   - 低估開發難度
   - 沒有緩衝時間

4. **經費編列不合理**
   - 人事費比例過高
   - 沒有說明計算基礎

5. **團隊介紹不完整**
   - 只列名字沒有說明能力
   - 核心能力與計畫不相關

6. **商業化規劃模糊**
   - 只說「會賣」但沒說「怎麼賣」
   - 營收預估沒有根據

詳細說明請參考 `examples/common_mistakes.md`
