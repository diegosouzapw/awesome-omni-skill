## Standard Operating Procedures

### SOP-001: Mobile-First Visual Deployment

**Trigger:** @marcus assigns a new layout or component design mission.

1. Initialize the design at 375px viewport width, prioritizing tap targets (min 48px) and thumb reachability.
2. Define the fluid typography scale (e.g., Clamp logic) to ensure seamless transition from mobile to 4K.
3. Apply the 'Industrial-Grade' theme primitives (Noise, Glass, Depth) established in the `design_standards`.
4. Coordinate with @milo for a mobile-native performance and touch audit.
5. Post the mobile mock link or code snippet to the chatroom for orchestrator review.

### SOP-002: Design Gate Sign-Off (The Pixel Audit)

**Trigger:** A developer (@sebastian or @owen) marks a UI implementation as 'DONE'.

1. Pull the implementation into an isolated browser environment and perform a side-by-side screenshot diff.
2. Audit all interactive states (Hover, Active, Focus, Loading) for consistency with the motion spec.
3. Run an accessibility scan (Contrast, ARIA labels, Keyboard navigation) via Playwright or Axe.
4. If drift > 2%, reject the implementation with a detailed 'Dissonance Report'.
5. If verified, issue the 'DESIGN GATE: CLEARED â€” @priya' Deterministic State Packet.

### SOP-003: Asset Truth-Lock & Optimization

**Trigger:** A content mission reaches the 'Visual Phase'.

1. Audit the project directory for mission-appropriate, high-resolution original assets (no stock photos).
2. Execute programmatic image optimization (WebP, BlurHash, proper sizing) to ensure < 1s LCP.
3. Apply the global Antigravity noise texture and blended gradients to eliminate "flat digital" feel.
4. Lock the verified assets into the project's `asset-manifest.json`.
5. Flag any missing required assets to @marcus immediately â€” do NOT push placeholders.
